## 2. How did you use AI as a teammate?

- Which AI tools did you use on this project (for example: ChatGPT, Gemini, Copilot)?
- Give one example of an AI suggestion that was correct (including what the AI suggested and how you verified the result).
- Give one example of an AI suggestion that was incorrect or misleading (including what the AI suggested and how you verified the result).

I used Claude Code to explain and improve my code. I could identify some errors, such as off-by-one errors in the attempt number and the higher/lower hint being inverted, but others were more difficult. Claude Code helped me identify the strange logic where the secret number was sometimes converted to a string, which caused issues with comparison in Python. It simply removed this part of the logic so that everything could be an integer. One part it got wrong initially was having the debug log update in time instead of lagging one guess behind. The initial fix made it draw after all other components and updates, which made it redraw in time but led to other issues due to other function calls like rerun or balloons (when you guess correctly). To fix this, I told it the issue and it showed me a solution where the debug box is declared above the game logic but drawn after it.

## 3. Debugging and testing your fixes

- How did you decide whether a bug was really fixed?
- Describe at least one test you ran (manual or using pytest)  
  and what it showed you about your code.
- Did AI help you design or understand any tests? How?

When determining whether a bug was fixed, I partially looked at the code, though I mostly compared the result when running the app to the intended result. The tests generated by Claude Code seemed simple enough that I maybe could have come up with them myself, which could mean that I have a pretty good understanding of coming up with test cases, or that I should have tried to prompt Claude Code to create more comprehensive tests, or that the current functions are simple enough that there aren't many edge cases you need to consider.
For example, the test file runs goes through what should be every case of update_score: increasing score on a win, the number of points earned decreasing with more attempts, the number of points earned being at minimum 10, losing points for guessing too high or too low, as well as some other cases such as modifying an existing score (rather than overwriting it), a nonsense or empty status value, and making sure the resulting score can be positive or negative.
